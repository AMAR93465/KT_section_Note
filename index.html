Pre Installation

Booting process means starting up a computer . It happens when you turn on computer and it loads the operating system.

Ø First your main OS isn't installed yet , So you need a bootable environment like WinPE.

Ø So this Windows pre-installation will do things like start installation process, positioning the hard drives (so the operating system knows where to store files, Keep things organized, tells windows where to go during installation, run installation tools.

Windows Pre-installation is a special setup before a actual windows installation.

In this booting process only we will provide what are the requirements we need to build OS. It is in default Plain OS without hardwares, without any applications. On top of that we will add applications, drivers, and what are the services to be enabled and disabled.

Example: if you are planning to paint your home , before paint you will clean the room and cover the things then prepare the wall .

Pre-install : Applications,BaseOS,Microsoft cumulative Update,PRE,Sys prep,firewall

Post Installation

Post-installation refers to everything that happens after you install the operating system (OS) especially Windows to get the system ready for actual use.

Post-installation makes a freshly installed OS usable, secure, and personalized.

We do install applications ,Create User Accounts, connect to network , install drives , configure security.

Post-install : Drives, images ,Post installation

Important Commands

RUN Windows Commands:

  • Cmd : Command Prompt
  • powershell : PowerShell
  • msconfig : System Configuration
  • services.msc : Services Manager
  • taskmgr : Task Manager
  • regedit : Registry Editor
  • firewall.cpl : Windows Defender Firewall (Control Panel)
  • wf.msc : Windows Defender Firewall with Advanced Security
  • explorer : Open File Explorer (Win + E)

Commands for CMD:

  • sc query : Shows all services and their statuses
  • sc query <servicename> : Check specific service
  • gpresult /R : Shows applied group policies for the current user and machine
  • gpresult /H C:\Users\Public\policy.html : Generates detailed report in HTML
  • wmic qfe list : Shows all installed updates (including cumulative updates)

OS Installation Flow (Based on Hardware)

  1. There are two types of hardware:
    • Legacy – N2FS
    • UEFI – BIOS – FAT32
  2. According to the hardware type, copy the bootable files and paste them into a USB drive.
  3. Insert the USB into the hardware system.
  4. Restart the PC. When the Philips logo appears, keep pressing the Boot key.
    (For Caronary tool, the key is F11)
  5. After entering the user and password, a popup will appear. Select the USB from that menu.
  6. The installation process will begin and complete successfully.

HIT Tool

HIT tool is a OS hardening tool through this we can build an Operating System .According the requirement we create the modules like drives ,applications etc.

HIT (Houston Installation Tool) is a Windows OS deployment tool, used for pre-installation and post-installation process. And also Automate and standardize the installation of Windows , Install device-specific drivers, software, and settings.

OS : which contains operating systems here we can check for any updates.

Packages : Here we can disable the keys if we want .

DRV (Drivers) : Here we can create a module like chip set drivers, Graphics, Audio drivers, USB drivers and Antivirus 3rd party drivers.

APP(Application) : We can install the applications like .NET, VC ++ and antivirus Applications. we can add default applications,and also 3rd party applications.

SCR: for creating configurations, PXE, Image filings, and Creating a page and adding a IP address all these we can do in this module.

QFE(cumulative Update): Every 2nd Tuesday they release new cumulative patch.

SRV(services) : Services in windows need to be in manual, automatic, enable or disable we can set these things .

TSK(task scheduling) : here we can schedule a task to perform . For example : if we want to restrict the firewall then we can schedule a task to restrict.

FWL : firewall means whatever the test coming from outside we will restrict it and if we need to go out to connect other devices we will make some restrictions.

POL(Polices) : it is like to add restrictions for example : if you want to create your own user login id, that can be used by others also within a limit .

DOD: Department of defense, every country have their own restrictions and policies depending upon them we set the DOD complaints.

TLS: configure what type of polices like what to open and not, user rights , user restrictions can be done here.

DPL : deployment model it is final step of OS , here we will add WinPE environment along with applications.
